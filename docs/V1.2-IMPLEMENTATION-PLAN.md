## Executive Summary

v1.2.0 transforms the simulator from a simple chat interface into a realistic consulting experience by adding:

1. **Rich client briefings** - Consultants get context before meetings
2. **Company logos** on selection screen (not faces)
3. **Person photos** in chat (talking to a human)
4. **Supabase database** for scenario data and session tracking
5. **Claude 3 Haiku** as the primary AI model (cost-effective)

---

## Goals

| Goal | Success Criteria |
|------|------------------|
| Pre-meeting context | Consultant sees company background before chat |
| Visual distinction | Logos on selection, photos in chat |
| Data persistence | Scenarios stored in database |
| Cost reduction | Switch to Claude 3 Haiku (~10x cheaper) |
| Session tracking | Store conversation history in database |

### SQL Schema

```sql
-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- ============================================
-- SCENARIOS TABLE (Denormalized for simplicity)
-- ============================================
CREATE TABLE scenarios (
  id TEXT PRIMARY KEY,  -- 'kindrell', 'panther', 'idm'
  
  -- Company Information
  company_name TEXT NOT NULL,
  company_logo_url TEXT,
  company_industry TEXT,
  company_tagline TEXT,
  company_background TEXT,  -- Rich text/markdown
  company_why_contacted TEXT,  -- Quote from company
  company_context JSONB DEFAULT '[]',  -- Array of bullet points
  
  -- Contact Information  
  contact_name TEXT NOT NULL,
  contact_role TEXT NOT NULL,
  contact_photo_url TEXT,
  contact_years_at_company INTEGER,
  contact_reports_to TEXT,
  contact_background TEXT,
  contact_communication_style TEXT,
  
  -- Scenario Configuration
  opening_line TEXT NOT NULL,
  system_prompt TEXT NOT NULL,
  difficulty TEXT DEFAULT 'medium' CHECK (difficulty IN ('easy', 'medium', 'hard')),
  max_turns INTEGER DEFAULT 15,
  
  -- Embedded Arrays (avoid extra tables)
  required_details JSONB DEFAULT '[]',
  hints JSONB DEFAULT '[]',
  
  -- Metadata
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- SESSIONS TABLE (Track user progress)
-- ============================================
CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  scenario_id TEXT REFERENCES scenarios(id) ON DELETE CASCADE,
  
  -- Timing
  started_at TIMESTAMPTZ DEFAULT NOW(),
  ended_at TIMESTAMPTZ,
  
  -- Progress
  turns_used INTEGER DEFAULT 0,
  details_obtained JSONB DEFAULT '[]',  -- Array of detail IDs
  hints_used JSONB DEFAULT '[]',  -- Array of hint IDs
  
  -- Status
  completed BOOLEAN DEFAULT FALSE,
  completion_percentage INTEGER DEFAULT 0
);

-- ============================================
-- MESSAGES TABLE (Conversation history)
-- ============================================
CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
  
  -- Message content
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system')),
  content TEXT NOT NULL,
  
  -- Tracking
  details_triggered JSONB DEFAULT '[]',  -- Which details this message revealed
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- INDEXES
-- ============================================
CREATE INDEX idx_sessions_scenario ON sessions(scenario_id);
CREATE INDEX idx_sessions_started ON sessions(started_at DESC);
CREATE INDEX idx_messages_session ON messages(session_id);
CREATE INDEX idx_messages_created ON messages(created_at);

-- ============================================
-- ROW LEVEL SECURITY (Optional - enable if adding auth)
-- ============================================
-- ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
-- ALTER TABLE messages ENABLE ROW LEVEL SECURITY;
```

## AI Configuration

### Model Selection

```typescript
// lib/ai-config.ts

export const AI_CONFIG = {
  // Primary model (cost-effective)
  primary: {
    provider: "openrouter",
    model: "anthropic/claude-3-haiku-20240307",
    maxTokens: 1000,
  },
  
  // Fallback if primary fails
  fallback: {
    provider: "openrouter", 
    model: "anthropic/claude-3.5-sonnet",
    maxTokens: 1000,
  },
  
  // Thinking delay for realism
  thinkingDelayMs: 800,
};
```

### API Route with Fallback

```typescript
// app/api/chat/route.ts

import { createOpenAI } from "@ai-sdk/openai";
import { streamText } from "ai";
import { AI_CONFIG } from "@/lib/ai-config";

export const maxDuration = 30;

const openrouter = createOpenAI({
  baseURL: "https://openrouter.ai/api/v1",
  apiKey: process.env.OPENROUTER_API_KEY,
});

export async function POST(req: Request) {
  try {
    const { messages, scenarioId } = await req.json();
    
    // Fetch scenario from Supabase
    const scenario = await getScenario(scenarioId);
    if (!scenario) {
      return new Response("Invalid scenario", { status: 400 });
    }
    
    // Artificial thinking delay
    await new Promise(resolve => setTimeout(resolve, AI_CONFIG.thinkingDelayMs));
    
    // Try primary model
    try {
      const result = await streamText({
        model: openrouter(AI_CONFIG.primary.model),
        system: scenario.system_prompt,
        messages,
        maxTokens: AI_CONFIG.primary.maxTokens,
      });
      return result.toDataStreamResponse();
    } catch (primaryError) {
      console.warn("Primary model failed, trying fallback:", primaryError);
      
      // Fallback to more capable model
      const result = await streamText({
        model: openrouter(AI_CONFIG.fallback.model),
        system: scenario.system_prompt,
        messages,
        maxTokens: AI_CONFIG.fallback.maxTokens,
      });
      return result.toDataStreamResponse();
    }
    
  } catch (error) {
    console.error("Chat API error:", error);
    return new Response("AI service unavailable", { status: 503 });
  }
}
```

| **Auth** | Built-in (can add later) |
| **Storage** | Built-in (for logos, docs) |
| **Realtime** | Not needed now, but available |
| **Cost** | Free tier sufficient for MVP |
| **Serverless** | Works great with Vercel |

### Why Denormalized Schema?

| Approach | Pros | Cons |
|----------|------|------|
| **Normalized (10 tables)** | Clean, DRY | Complex queries, slow for reads |
| **Denormalized (3 tables)** | Fast reads, simple | Some duplication |

**Decision:** Start denormalized. Can normalize later if we need multi-contact per company.

### Why Claude 3 Haiku?

| Model | Quality | Cost | Decision |
|-------|---------|------|----------|
| Claude 3.5 Sonnet | Excellent | ~$3/1M tokens | Too expensive for training tool |
| Claude 3 Haiku | Good | ~$0.25/1M tokens | Best value for character roleplay |
| GPT-4o Mini | Good | ~$0.15/1M tokens | Alternative option |
| Llama 3.1 | Decent | Very cheap | May not maintain character well |

**Decision:** Haiku primary, Sonnet fallback.
